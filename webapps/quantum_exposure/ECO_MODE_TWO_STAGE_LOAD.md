# ECO Mode Two-Stage Data Loading Optimization

## Overview
Implemented two-stage data loading for ECO mode (lite client) to dramatically accelerate initial dashboard load while keeping the site fully navigable during full data fetch.

## Architecture

### Phase 1: Ultra-Fast Initial Load
1. Load `dashboard_snapshot_meta.csv` and `dashboard_pubkeys_aggregates.csv` (metadata)
2. Render chart and KPI context immediately
3. Load lightweight `dashboard_pubkeys_ge_1btc_top50.csv` (~5.3 KB)
4. Populate top exposures panel with top 50 addresses
5. Return control to user (site is interactive)

### Phase 2: Background Full Data Load (Non-blocking)
1. Start fetching full `dashboard_pubkeys_ge_1btc.csv` (~16 MB) in background
2. User can navigate, filter, and interact with top 50 data
3. When full load completes, silently swap data and update UI
4. User gets full filtering capability without interruption

## Performance Benefits

### File Size Reduction
- **Top 50 version**: 5.3 KB
- **Full version**: ~16 MB
- **Reduction**: ~3000x smaller initial payload

### Load Timeline
```
Traditional ECO Mode:
[metadata] → [16 MB full ge1] → [UI ready to sort/filter] ≈ slow

New Two-Stage ECO Mode:
[metadata] → [5.3 KB top50] → [UI ready immediately]
                    └─ [background 16 MB] → [seamless data swap]
                    Time to interactive: ~100x faster
```

## File Changes

### New Data Files
- `webapps/quantum_exposure/webapp_data/*/dashboard_pubkeys_ge_1btc_top50.csv`
  - 50 rows of highest-value addresses per snapshot
  - Pre-sorted by exposed supply (descending)
  - Generated by `generate_top50_eco_csvs.py`

### Code Changes
1. **main.js - State Variables**
   - `ge1FullDataLoadAbortController`: Signal to cancel background load if user navigates
   - `ge1FullDataLoadPromise`: Promise tracking full data load
   - `ge1IsUsingTop50`: Boolean flag indicating current data source

2. **main.js - loadSnapshotData() Function**
   - Phase 2a: Load and display top_50 immediately
   - Phase 2b: Spawn background fetch for full data
   - Intelligent swap when full load completes
   - Cache updated with full data for future visits

3. **generate_top50_eco_csvs.py**
   - Utility script to generate top_50 files from full ge1 CSVs
   - Sorts by summed exposed_supply_sats across script types
   - Maintains full CSV structure for seamless parsing

## Technical Details

### Abort Signal Handling
- Previous background loads are aborted if user navigates to different snapshot
- Prevents stale data swaps from old loads corrupting current view

### Cache Strategy
- Initially caches top_50 data (fast subsequent load)
- Updates cache to full data when background load completes
- Second visit to same snapshot loads from cache (instant + non-blocking background swap)

### Safety Checks
- Verifies snapshot hasn't changed before swapping data
- Gracefully handles network errors in background load
- Maintains filter cache clearing to prevent stale filtered results
